How It's Made

Photogrammetry

The model of my dad was made using 123D Catch, a cloud-based photogrammetry mobile application. Although the application is limited in its ability to create detailed 3D models, much of the detail is lost via model post processing and 3D printing post processing, so this was not a huge issue for me. I started by taking various pictures of my dad at various angles and letting the application stitch all the pictures together to create a .STL file. Due to poor lighting issues, only half of the model came out well. To fix this problem, I deleted the bad half and mirrored the good half in Autodesk’s Meshmixer. The intersection between the two halves was smoothened out as well. Since hollow shells are not able to be sliced for printing, the resulting model made solid using the “Inspector” tool and repairing all holes. This fills any holes, especially the hole caused by missing data inside the neck of the model. The base of the model was then extruded to look a little more like a bust. The model was then sliced, printed, and sanded to create the final product.

Unfortunately I do not have any of the original files or other pictures of the model :(

Scanning Using Kinect Sensor

I also experimented with using a scanning technique instead of photogrammetry to generate 3D models. The Xbox 360 Kinect sensor has an included infrared sensor which allows it to be used as a cheap alternative to pricey scanners. However, the Kinect is old technology and the picture quality is not the best. For software, I tried using the free version of Skanect, however, it limits the number of polygons that can be exported for a model. The polygon count is capped at 5000 which is very little for the level of detail I wanted. I was able to make a 3D model of myself using this method, however, the low polygon export removes a lot of the detail.
